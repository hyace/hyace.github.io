Git时光机

处于对theme的追求，找到了这个页面,恨不能把列出来的都试一遍，但有
的很好看的主题装起来比较麻烦，还有很多依赖或者配置，下午就经历了一次，
不知道是哪一个主题，装完以后我的主页上就没有article显示了，我就想用git
回滚到没装主题的下午两点。
但是之前用git也就是add commit pull push流，知道git强大到随心所欲，但
就是无从下手。研究很久之后尝试，先checkout到了两点的commit。

git checkout [SHA]  
git reset
git status

然后就列出了红红一片的unstaged，但是我想的是要把本地的更改完全撤销，这
些文件就不知道如何处理了。

后来干脆休克疗法…把octopress文件夹下除了source里的_posts都删掉，从仓
库里checkout。尝试了一下如下命令：

git checkout -- _config.yml  
git status

然后列出的红字里就没有_config.yml了。
但是这数百个文件要改到何时…想起了很久没用过了的shell脚本，把这几百行
的输出都复制到sos.sh里，每行前加上git checkout --，保存，./sos.sh
运行。果然都消失了。然后赶紧恢复到github。

git push origin source

问题是解决了，但是做的很不好，终究还是对git不熟悉，有点误打误撞的成分，
看来还是要好好研究下。
